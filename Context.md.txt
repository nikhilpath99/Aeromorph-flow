I want you to help me build a Python/PyTorch research codebase for a project called **AeroMorph-Flow**.

The project goal is to build a **retrieval-augmented physics-memory model for morphing airfoil flow prediction**.

The first MVP should use:

* Python
* PyTorch
* NumPy
* SciPy if needed
* scikit-learn if needed
* matplotlib
* pandas
* optional XFOIL integration, but the code should still run with synthetic/mock data if XFOIL is not installed

Main idea:

Instead of training only:

`airfoil geometry + AoA + Re → Cl, Cd, Cp`

I want to train a model on morphing transitions:

`airfoil_before + airfoil_after + AoA + Re + flow_before → flow_after`

or:

`delta_geometry + AoA + Re + Cp_before → delta_Cp, delta_Cl, delta_Cd`

The model should learn how flow changes when the airfoil geometry morphs.

Build the project in phases.

2. Implement NACA 4-digit airfoil generation.

Functions needed:

```python
generate_naca4(code: str, n_points: int = 200) -> dict
```

Return:

* x coordinates
* upper surface y
* lower surface y
* closed airfoil coordinates
* metadata: camber, camber_position, thickness

3. Implement airfoil morphing.

Function:

```python
morph_airfoils(airfoil_a, airfoil_b, n_steps: int) -> list
```

It should produce smooth intermediate airfoils using coordinate interpolation.

Each morphing path should look like:

```text
airfoil_0 → airfoil_1 → airfoil_2 → ... → airfoil_n
```

4. Implement a mock aerodynamic solver first.

Because XFOIL may not be installed, create a mock solver that returns plausible synthetic values for:

* Cp distribution
* Cl
* Cd
* Cm

The mock solver does not need to be physically perfect. It should allow the ML pipeline to run end-to-end.

Function:

```python
solve_airfoil_mock(airfoil, aoa_deg: float, reynolds: float) -> dict
```

Return:

* cp: array of length n_points
* cl: float
* cd: float
* cm: float
* converged: bool

Later, create an XFOIL runner wrapper but keep it optional.

6. Implement baseline model.

Baseline:

```text
geometry_after + AoA + Re → Cp_after, Cl_after, Cd_after
```

Use a simple PyTorch MLP.

7. Implement delta model.

Delta model:

```text
geometry_before + delta_geometry + AoA + Re + Cp_before → delta_Cp, delta_Cl, delta_Cd
```

Then reconstruct:

```text
Cp_after = Cp_before + delta_Cp
Cl_after = Cl_before + delta_Cl
Cd_after = Cd_before + delta_Cd
```

8. Implement memory bank.

Each memory item stores:

* embedding
* delta_geometry
* AoA
* Re
* delta_Cp
* delta_Cl
* delta_Cd
* metadata

Use nearest-neighbor retrieval first with scikit-learn or pure NumPy.

Similarity should initially be based on:

* delta_geometry embedding
* AoA
* log(Re)
* maybe Cp_before embedding

9. Implement retrieval-augmented model.

For each query transition:

* compute query embedding
* retrieve top-k similar memory items
* aggregate retrieved memory features
* feed query features + retrieved features into MLP
* predict delta_Cp, delta_Cl, delta_Cd

Start simple:

* mean pooling over top-k retrieved memory features
* later attention can be added
